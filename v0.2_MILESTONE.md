# Plum Notes v0.2: The Sync Engine

**Status:** Planned
**Focus:** Multi-device sync, row-level granularity, and mobile companion app.

This milestone represents the architectural shift from a single-device JSON blob store to a distributed, multi-device capable platform powered by CRDTs.

## Priority 1: Schema Migration & Normalization
* **Goal:** Move from a monolithic JSON blob to a normalized, row-per-note SQLite schema.
* **Scope:**
  * Draft the schema migration plan (must be flawless to avoid data loss).
  * Update `db.rs` migration runner.
  * Extract `Note` fields (id, title, categoryId, etc.) into proper columns.
  * Extract `tags` into a `NoteTags` join table.
  * Update all React state management to handle granular row updates instead of monolithic array replacements.

## Priority 2: CR-SQLite Integration
* **Goal:** Enable masterless, peer-to-peer sync using CRDTs.
* **Scope:**
  * Integrate the `crsqlite` extension into the Tauri SQLite backend.
  * Set up the CRDT conflict resolution layer.
  * Build the sync protocol (likely via a lightweight relay server or direct P2P).

## Priority 3: React Native Mobile Companion
* **Goal:** Frictionless quick capture from iOS and Android.
* **Scope:**
  * Initialize React Native (Expo) project.
  * Port the CR-SQLite data layer to React Native.
  * Build the mobile Quick Capture UI.
  * Implement the sync protocol to communicate with the desktop client.
